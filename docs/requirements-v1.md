# v1 要件定義 — 流域単位の自然資本算定・可視化

最終更新: 2026-07-02
ステータス: ドラフト（実装開始の基準線）

---

## 0. このドキュメントの位置づけ

企画書（`nature_city_platform_v2.pptx`）が描く壮大な構想（5 モジュール A〜E）のうち、
**最初に作る最小スライス**を実装可能な粒度で定義する。投資家向け資料ではなく、
開発者（当面は個人開発）が手を動かすための基準線。

対象は企画書の **MODULE A（生態系・流域モニタリング）** と
**MODULE B（自然資本価値算定）** の交差部分。景観スコアリング（C）・人口×インフラ（D）・
自然回復プランナー（E）・TNFD レポートは **v1 スコープ外**（後述のロードマップ参照）。

---

## 1. v1 のゴール（1 文）

> 対象地域を指定すると、地形 DEM から流域（集水域）を導出し、土地被覆を分類し、
> 流域ごとの自然資本（生物物理量）を算定して、地形が読める地図上に重ねて表示する。

「鴨川流域を選ぶ → 地形と集水域が描かれ → 流域ごとの炭素・緑被・保水が数値で出る」
までを一気通貫で成立させる。

---

## 2. スコープ

### 2.1 やること（In Scope）

1. **流域導出**: 基盤地図情報 DEM から、窪地補正 → 流向 → 流量集積 → 集水域ポリゴンを生成。
2. **土地被覆**: ESA WorldCover 10m を対象地域でクリップし、被覆クラスへ整理。
3. **自然資本算定（生物物理量）**: 流域ポリゴンを集計単位に、以下を算定。
   - 土地被覆構成比（森林・草地・農地・水域・市街地・裸地 の面積と割合）
   - 緑被率 / NDVI（Sentinel-2 由来、可能なら季節代表値）
   - 炭素蓄積量（InVEST Carbon: 土地被覆別係数ルックアップ）
   - 流域保水の代理指標（InVEST 由来の水収量/保水、または被覆・土壌からの近似）
4. **可視化（地図）**: 陰影起伏の上に、流域ポリゴンと自然資本レイヤーをコロプレスで表示。
   流域をクリックすると詳細パネルに数値を表示。
5. **任意地域対応**: 対象地域は設定で差し替え可能な作りにする（最初のテスト対象 = 鴨川流域）。

### 2.2 やらないこと（Out of Scope / v1.1 以降）

- 金額換算（円）— 換算係数の妥当性検証が必要なため v1.1 以降。
- 景観スコアリング（写真 → スコア）= MODULE C。
- 人口動態 × インフラ可視化 = MODULE D。
- 自然回復候補地プランナー = MODULE E。
- TNFD 準拠レポートの PDF 生成。
- ユーザーによる自前データのアップロード・上書き。
- 任意地域を**その場で動的計算**する機能（v1 は事前計算した地域のみ）。
- 認証・課金・マルチテナント。

## 3. 想定ユーザーと使い方（v1）

個人開発フェーズのため、まずは**開発者自身と初期ヒアリング相手（建築・ランドスケープ関係者、
自治体環境担当）にデモを見せる**ことがゴール。ログインなしで閲覧できる公開デモを想定。

代表シナリオ:
- 「この流域は森林が何 % で、炭素をどれだけ蓄えていて、保水力はどの程度か」を地図で一望する。
- 隣接流域と比較し、自然資本が高い/低いエリアの当たりをつける。

## 4. データソース（詳細は `data-sources.md`）

| データ | ソース | 解像度/粒度 | 用途 |
|---|---|---|---|
| DEM（標高） | 国土地理院 基盤地図情報 数値標高モデル | 5m / 10m | 流域導出・陰影起伏 |
| 土地被覆 | ESA WorldCover | 10m | 被覆構成・炭素・保水の入力 |
| 植生指標 | Copernicus / Sentinel-2（NDVI） | 10m | 緑被率・NDVI |
| 河川 | 国土数値情報 河川データ | ベクタ | 流域の妥当性確認・水系ラベル |
| 自然資本係数 | InVEST モデル付属テーブル + 文献 | — | 炭素蓄積・保水の係数 |

すべて公開データ。ライセンス・出典表記は `data-sources.md` で管理する。

## 5. 処理パイプライン（バッチ / Python）

対象地域ごとに一度だけ実行し、成果物（タイル・統計）を生成する。常時稼働不要。

```
[入力] 対象地域の範囲（バウンディングボックス or 行政界 or 主要河川の水系）
   │
   ├─ 1. DEM 取得・モザイク・投影統一（rasterio）
   │
   ├─ 2. 流域導出（pysheds）
   │      窪地補正 → 流向(D8) → 流量集積 → 集水域ポリゴン化（geopandas）
   │
   ├─ 3. 土地被覆クリップ・整理（WorldCover → クラス統合）
   │
   ├─ 4. 植生指標（Sentinel-2 → NDVI 合成、緑被率算出）
   │
   ├─ 5. 自然資本算定（natcap.invest: Carbon / Water Yield）
   │      流域ポリゴンでゾーン統計 → 流域別サマリ
   │
   ├─ 6. 集計（流域 × 指標の統計テーブル / JSON）
   │
   └─ 7. タイル化
          陰影起伏: gdaldem hillshade → ラスタ PMTiles
          流域+統計: tippecanoe → ベクタ PMTiles
```

成果物:
- `hillshade.pmtiles`（地形）
- `watersheds.pmtiles`（流域ポリゴン＋属性）
- `stats/<region>.json`（流域別の自然資本サマリ）

これらを Cloudflare R2 に配置し、Web から直接読む（タイルサーバー不要）。

## 6. 画面要件（Web / Next.js + MapLibre）

参照イメージ: **YAMAP の流域地図**（地形が読める陰影 + 流域の色分け）。

### 6.1 レイアウト
- 全画面地図（MapLibre GL JS）。
- 左または下にサイドパネル（選択中の流域の自然資本詳細）。
- レイヤ切替（陰影 / 流域 / 自然資本指標の選択）。

### 6.2 地図レイヤ（下から）
1. 陰影起伏（ラスタ PMTiles）＋ 任意で等高線・尾根線。
2. 流域ポリゴン境界。
3. 自然資本コロプレス（指標を 1 つ選んで塗り分け: 炭素密度 / 緑被率 / 保水 など）。

### 6.3 インタラクション
- 流域をクリック → パネルに指標一覧（土地被覆構成の円/帯グラフ、炭素蓄積、緑被率、保水）。
- 指標セレクタでコロプレスの塗り分け対象を切替。
- ホバーで流域名・水系名のツールチップ。

### 6.4 見た目の基準
- 地形の可読性を最優先（YAMAP 的）。彩度は抑え、自然資本レイヤーは半透明で重ねる。
- データ可視化の配色は別途デザインガイドに従う（連続量は単一色相のシーケンシャル、
  増減がある指標は発散配色）。ライト/ダーク両対応を前提に設計する。

## 7. 非機能要件

| 項目 | v1 の基準 |
|---|---|
| 配信 | 静的（PMTiles + JSON）。CDN 経由。サーバープロセスなし。 |
| コスト | 個人開発として月額をほぼゼロに（R2 エグレス無料・Vercel Hobby）。 |
| パフォーマンス | 初期表示 3 秒以内（対象地域 1 つ、タイルは範囲・ズーム差分読み）。 |
| 再現性 | パイプラインはコード＋設定から再実行で同一成果物を生成できる。 |
| 移植性 | 動的化（v1.1）時に静的タイルはそのまま流用できる分離設計。 |
| ライセンス順守 | 各データの出典・帰属表記を UI のクレジットに表示。 |

## 8. 精度・品質 KPI（v1 の合格ライン）

自治体調達では必ず精度を問われるため、v1 の段階から目標値を持つ。

| 指標 | 目標 | 検証方法 |
|---|---|---|
| 流域境界の妥当性 | 主要河川の水系と矛盾しない（目視 + 河川データ重ね合わせ） | 国土数値情報 河川データと重畳確認 |
| 土地被覆の一致 | WorldCover クラスを正しく集計（面積の保存） | クリップ前後の面積突合 |
| 炭素蓄積の桁 | 既存文献・自治体公表値とオーダーが一致 | 文献値レンジ内か確認 |
| 再現性 | 同一入力で成果物が一致 | パイプライン再実行での差分ゼロ |

数値そのものより、**「出典があり・再現でき・オーダーが妥当」**を v1 の品質基準とする。

## 9. 未解決論点（実装中に決める）

- 集水域の**分割粒度**（最小集水面積のしきい値）。都市部で細かすぎ/粗すぎを調整。
- 保水指標に InVEST Water Yield を使うか、被覆・土壌からの簡易近似に留めるか。
- NDVI の**季節代表**の取り方（単月 / 年間合成 / 生育期）。
- 炭素係数の出典（InVEST 既定 vs 日本の森林簿・J-クレジット係数）。
- コロプレスの**階級区分**（等間隔 / 分位 / Jenks）。

## 10. この先のロードマップ（v1 完了後）

1. **v1.1**: 任意地域のその場計算（FastAPI + TiTiler + PostGIS を Fly.io）／自前データ上書き。
2. **v1.2**: 自然資本の**金額換算**（係数の妥当性検証込み）。
3. **MODULE C**: 景観スコアリング。
4. **MODULE D**: 人口動態 × インフラ可視化。
5. **MODULE E**: 自然回復候補地プランナー（A〜D 統合）。
6. **レポート**: TNFD LEAP へのマッピング／PDF 生成。
   → v1 の生物物理量は LEAP 指標へマップできる形で設計し、手戻りを避ける。
